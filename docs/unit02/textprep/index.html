The big picture

From digital soup to linguistic meaning

Text preprocessing is everything that happens above the dotted line that most NLP courses skip past. Four dependencies decide what counts as "correct" here — and every later stage (grammar checking, translation, sentiment analysis, chatbots) inherits whatever mistakes get made in this pipeline.

Stage 1

💾 Bits

A raw sequence of 0s and 1s in a file. Pure storage — the machine doesn't yet know these bits mean "N", "L", "P".

Stage 2

🔤 Characters

Bits get mapped to graphemes via a character encoding like UTF‑8. Now we have letters.

Stage 3

🧩 Tokens

Characters are grouped into words — this grouping process is called tokenization.

Stage 4

📄 Sentences

Tokens are grouped into sentences — this is sentence segmentation.

Worked example: "NLP is amazing. It helps students."

Three examples of the same idea, so you see the pattern, not just one instance:

Example A

Input text: "cats" → as bits it's an unreadable byte string; as characters "c‑a‑t‑s"; as one token cats; inside its own one-word sentence.

Example B

Input text: "I'm fine." → bits → characters → tokens ["I'm","fine","."] → one sentence, showing tokenization must decide whether I'm splits into two.

Example C

Input text: "NLP is amazing. It helps students." → tokens ["NLP","is","amazing",".","It","helps","students","."] → two sentences, split at the first period.

The same text splits differently depending on what the system needs.

I'm

A parser needs grammar features, so it's expanded: I + am — two tokens.

$300

A text-to-speech system needs it spoken: three / hundred / dollars — three tokens.

governor's

Pattern matching

Regular expressions — the tool NLP has leaned on since its first chatbot

A regular expression (regex) is a formal pattern language for describing sets of strings — it lets code say "match anything that looks like this shape" instead of listing every possible word. It is the oldest and still one of the most useful tools in text preprocessing: finding abbreviations, splitting sentences, pulling out dates, validating input.

What is ELIZA, and why does every NLP course mention it?

ELIZA was written in 1964–66 by Joseph Weizenbaum at MIT, and is widely considered the first chatbot. It is named after Eliza Doolittle, the character in George Bernard Shaw's play Pygmalion who is taught to speak differently — a nod to how the program "speaks" back to whoever types to it. ELIZA's most famous personality, DOCTOR, imitated a Rogerian psychotherapist. It never understood a single sentence; it simply used a library of regular-expression rules to spot a pattern in your sentence, then reflected fragments of your own words back at you as a question. "I need some help" matches a rule for I need (.*) and becomes "What would it mean to you if you got some help?" The trick — and the reason it fooled people — is that pattern matching plus reflection can feel like understanding, even with zero comprehension underneath.

Tokenization

What is a word? A deceptively hard question

Splitting text into individual words or tokens is essential for every NLP task — but four recurring challenges make it genuinely tricky.

1

"Hey!! How are you? :)"

Punctuation and emoticons — are they tokens too?

2

"I do uh main- mainly business…"

Disfluencies and false starts — what counts as a word?

3

"They're studying…"

Contractions hide two words inside one.

4

"cats" vs. "cat"

Same root, different forms — this is exactly what stemming/lemmatization later normalize.

One character, three jobs: the period

"Clairson International Corp. said it expects to report a net loss of $3.9 to $4 million for its year ending Sept. 24."
Wall Street Journal, 1988

Decimal point

$3.9 — the period is part of the number itself.

Abbreviation marker

Corp. and Sept. — the period marks a shortened word.

Sentence boundary

The final period after "24" — and this time it isn't a decimal point.

German compounding

Lebensversicherung

= Leben (life) + Versicherung (insurance), written as one word but holding two concepts. Also: Haustür = Haus + Tür (house door).

Turkish agglutination

çöplüklerizdekilerdenmiydi

One word → "was it from those that were in our garbage cans?" Root + plural + possession + location + tense, all suffixed on.

End-of-line hyphens

govern-\nment

A line-break hyphen, not a real word boundary. About 5% of hyphens in a real English corpus were purely line-break artifacts.

Idiom

"in spite of"

Means exactly "despite" — three tokens that should behave like one.

Dates & money

Nov. 18, 1989

= 18 November 1989 = 11/18/89 — all need normalizing to the same value.

Ambiguous scope

"no one"

= nobody, one idea — but in "No one man can do it alone," it's genuinely two words.

2Backward matching

thetabledownthere, read right to left, finds: there → down → table → the. Produces the correct segmentation here.

3Bidirectional

Run both, compare

Forward AND backward matching, then choose whichever segmentation looks better — used by many modern systems.

4Character-as-word

Treat every character as a token

我喜欢学习 → 我 | 喜 | 欢 | 学 | 习. Works surprisingly well (most Chinese words are 1–2 chars) but splits real multi-char words like 喜欢 ("like").

5Statistical

Learn co-occurrence

北京大学 → 北京 | 大学 (Beijing | University). More accurate, handles new vocabulary — needs large training corpora.

6Lexical + hybrid

Dictionaries + rules + stats

私は学生です → 私 | は | 学生 | です ("I am a student"). Solved by combining dictionary lookup, grammar rules, and statistics — as JUMAN and ChaSen do for Japanese.

Subword tokenization

Byte-Pair Encoding: learning a vocabulary instead of hand-defining one

The problem BPE solves: how do we handle words we've never seen before? Instead of words or characters, BPE induces "subword" tokens automatically from data — so lower can be represented as low + er, even if "lower" itself was never in the training set.

1Initialize — start with characters
2Find the most frequent adjacent pair
3Merge it into one new symbol
4Repeat until k merges are done

Sentence segmentation

What even counts as "one sentence"?

Obvious rule

"Period + space + capital letter"

Works fine: "The lecture ended. Students left." Fails here: "The meeting is on Sept. 15." — the period belongs to the abbreviation, not the sentence end.

How often does it work?

Accuracy depends entirely on the corpus

98% on The Call of the Wild (literary, well-behaved) vs. only 88.4% on the Wall Street Journal — where 47% of periods are abbreviation-markers, not sentence ends.

MarkJobBehavior
.PeriodUsually ends a sentence — but not always
?Question markReliably ends a question
!ExclamationReliably ends an exclamatory sentence
;SemicolonConnects related clauses — depends on task
,CommaNormally doesn't end a sentence, but some applications split long clauses here

Clue 1

Case of surrounding words — is the next word capitalized because it's a new sentence, or a proper noun?

Clue 2

Part of speech nearby — a proper noun after "Mr." suggests abbreviation, not boundary.

Clue 3

Word length before/after — very short words ending in a period (Dr., Mr., Jr.) are often abbreviations.

Clue 4

Known abbreviation list — a dictionary of common abbreviations: Dr., Prof., Corp., Inc., Sept.

Clue 5

Prefixes and suffixes — "International Corp." — the suffix strongly signals abbreviation.

Garden path

"…Penn St. University President…" reads as one sentence. Add "announced" after it and it suddenly becomes two.

Rule-based — ALEMBIC

75 abbreviations + 100+ hand rules, built over ~6 staff-months. 99.1% accuracy on WSJ text.

Trainable — SATZ / Riley

Trainable in minutes on under 1,000 sentences. 98–99.8% accuracy — robust to OCR errors.

Plot twist: Alembic + Satz together score 99.5% — higher than either alone. Hybrid systems consistently win.

Vocabulary check — six terms worth knowing cold

Text normalization

Transforming text into a more uniform format for easier processing.

Lemma

The base form of a word — lemmatization reduces "running" to "run".

Corpus

A structured collection of texts, used as a dataset for NLP tasks.

Token

A single, meaningful element in text — tokenization splits text into tokens.

Training / test corpus

One dataset teaches the model patterns; a separate one evaluates performance.

Vocabulary

The set of unique tokens a model uses for processing.

Your turn

Out-tokenize the tokenizer

For each puzzle: how many tokens? Where's the sentence boundary? Think first, then reveal.

Puzzle 1

"…headquarters at 55 Water St. A spokesman declined…"

One sentence — or two? "St." = Street here, so the period is an abbreviation marker and, in this case, also happens to sit right before a genuine sentence boundary. Context (a spokesman being a new clause) is the only way to tell.
Puzzle 2

"Boston-based"

One token, or two words joined by a hyphen? Most modern tokenizers keep it as one token (a compound adjective) — but a parser might split it into "Boston" + "based" for grammar analysis.
Puzzle 3

"I.D.'s" and "1980's"

Plural, or possessive? "I.D.'s" is usually a plural (multiple ID cards); "1980's" is usually a decade plural too, not a possessive — yet the apostrophe looks identical to "governor's" in Dependency 4. Only convention (and style guide) decides.
